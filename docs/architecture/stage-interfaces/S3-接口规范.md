# S3 接口规范（2026-05-21 至 2026-06-20）

## 1. 阶段目标

S3 目标是打通“提取-判定-复核”核心链路，重点强化 rules_engine、model_review、evidence，产出条款级 PASS/FAIL/REVIEW 结果。

## 2. 数据结构

### 2.1 RuleDefinition

```json
{
  "rule_id": "R-RSA-001",
  "field": "crypto.rsa.key_length",
  "operator": ">=",
  "threshold": "3072",
  "priority": "P0",
  "review_on_equal": true
}
```

### 2.2 RuleResult

```json
{
  "rule_id": "R-RSA-001",
  "standard_id": "S1-1.2",
  "check_item": "不合适的加密算法",
  "field": "crypto.rsa.key_length",
  "value": "2048",
  "verdict": "REVIEW",
  "reason": "RSA 2048 requires manual review by baseline policy",
  "confidence": 0.9
}
```

### 2.3 ReviewItem

```json
{
  "rule_id": "R-RSA-001",
  "text": "RSA2048",
  "context": "TLS config screenshot"
}
```

### 2.4 EvidenceTraceV3

```json
{
  "rule_id": "R-RSA-001",
  "field": "crypto.rsa.key_length",
  "value": "2048",
  "page": 3,
  "snippet": "RSA2048",
  "image_ref": "img-0001",
  "source_type": "image_ocr"
}
```

## 3. 模块接口

## 3.1 rules_engine

### evaluate_rules(fields: dict[str, str], rules: list[RuleDefinition]) -> list[RuleResult]

输入：
- fields: dict[string, string]
- rules: list[RuleDefinition]

输出：list[RuleResult]

异常：
- TypeError（已有）
- ValueError（建议：规则字段缺失）

### classify_review_items(results: list[RuleResult]) -> list[ReviewItem]

输入：
- results: list[RuleResult]

输出：
- verdict == REVIEW 的规则项列表

## 3.2 model_review

### semantic_review(item: dict[str, str]) -> dict[str, str]

S3扩展建议字段：
- normalized: string
- explanation: string
- confidence: string
- policy_hint: string（建议新增）

### batch_semantic_review(items: list[ReviewItem]) -> list[dict[str, str]]

输入：
- items: list[ReviewItem]

输出：
- 与输入等长的复核结果列表

## 3.3 evidence

### build_evidence_trace(hit: dict[str, str]) -> EvidenceTraceV3

输入：
- hit 建议至少包含 field/value/page/snippet/image_ref

输出：EvidenceTraceV3

### build_evidence_batch(hits: list[dict[str, str]]) -> list[EvidenceTraceV3]

输入：
- hits: list

输出：list[EvidenceTraceV3]

## 3.4 reporting

### export_summary(results: list[RuleResult]) -> dict[str, int]

输入：
- results: list[RuleResult]

输出：
- PASS/FAIL/REVIEW 统计

### export_analysis_report(task_id: str, summary: dict[str, int], rules: list[RuleResult], evidence: list[EvidenceTraceV3]) -> dict[str, object]

输出示例：
```json
{
  "task_id": "task-0001",
  "risk_level": "medium",
  "summary": {
    "PASS": 8,
    "FAIL": 1,
    "REVIEW": 3
  },
  "highlights": [
    "Weak algorithm MD5 found",
    "RSA2048 requires review"
  ],
  "findings": [
    {
      "standard_id": "S1-1.2",
      "check_item": "不合适的加密算法",
      "verdict": "REVIEW"
    }
  ]
}
```

## 4. S3 主链路结果结构

```json
{
  "task_id": "task-0001",
  "fields": {
    "crypto.rsa.key_length": "2048",
    "crypto.tls.version": "1.2"
  },
  "rule_results": [
    {
      "rule_id": "R-RSA-001",
      "verdict": "REVIEW"
    }
  ],
  "review_results": [
    {
      "normalized": "RSA-2048",
      "confidence": "0.90"
    }
  ],
  "evidence": [
    {
      "rule_id": "R-RSA-001",
      "page": 3,
      "image_ref": "img-0001"
    }
  ]
}
```

## 5. 兼容性要求

- S2 的 StructuredField 不可删除，S3 仅扩展。
- 规则结果必须包含 verdict，缺失默认 REVIEW。
- 单条 `FAIL` 只表示对应标准条款不符合，不表示整份报告失败。
