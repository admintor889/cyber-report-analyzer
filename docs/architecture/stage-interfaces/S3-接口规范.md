# S3 接口规范（条款级 finding 版）

## 1. 阶段目标

S3 负责把结构化字段接入主规则库，完成两份清单的统一建模、条款级判定、复核解释和证据追溯。输出单位是 `RuleResult`、`ReviewItem`、`EvidenceTraceV3` 和 `Finding`。

## 2. 数据结构

### 2.1 RuleDefinition

```json
{
  "rule_id": "R-RSA-001",
  "standard_id": "S1-1.2",
  "check_item": "不合适的加密算法",
  "field": "crypto.rsa.key_length",
  "operator": ">=",
  "threshold": "3072",
  "priority": "P0",
  "rule_version": "2026.06",
  "source_refs": [
    {
      "source_name": "Cyber_Certificate_checklist.xlsx",
      "location": "sheet1!B12"
    }
  ]
}
```

## 3. 模块接口

### 3.1 rules_engine

#### evaluate_rules(fields: dict[str, str], rules: list[RuleDefinition]) -> list[RuleResult]

- 规则字段缺失建议抛出 `ValueError`。
- 结果必须包含 `verdict`、`reason`、`paragraph_id`。

#### classify_review_items(results: list[RuleResult]) -> list[ReviewItem]

#### build_findings(results: list[RuleResult]) -> list[Finding]

### 3.2 model_review

#### semantic_review(item: dict[str, str]) -> dict[str, str]

- 仅负责归一化、解释和复核建议。
- 不得覆盖规则 verdict。

#### batch_semantic_review(items: list[ReviewItem]) -> list[dict[str, str]]

### 3.3 evidence

#### build_evidence_trace(hit: dict[str, str]) -> EvidenceTraceV3
#### build_evidence_batch(hits: list[dict[str, str]]) -> list[EvidenceTraceV3]

### 3.4 reporting

#### export_summary(results: list[RuleResult]) -> dict[str, int]
#### export_analysis_report(task_id: str, summary: dict[str, int], findings: list[Finding], evidence: list[EvidenceTraceV3]) -> dict[str, object]

## 4. 兼容性要求

- S2 的 StructuredField 不可删除。
- 规则结果必须包含 verdict，缺失时默认 `REVIEW`。
- 单条 `FAIL` 只表示对应标准条款不符合，不表示整份报告失败。
- 主规则库版本和来源映射必须可回放。

## 5. 退出标准

S3 结束时，S4 必须可以直接使用 `Finding` 和 `EvidenceTraceV3` 生成页面和 Word 导出，不得再回到原始字段层重新计算条款结论。
