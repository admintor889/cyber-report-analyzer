# S4 接口规范（2026-06-21 至 2026-07-10）

## 1. 阶段目标

S4 目标是完成系统集成，支撑“上传、查看、筛选、证据查看、导出”完整流程，并保证页面与导出的条款级结果一致。

## 2. 数据结构

### 2.1 QueryFilter

```json
{
  "verdict": "REVIEW",
  "field": "crypto.rsa.key_length",
  "page": 3,
  "keyword": "RSA"
}
```

字段类型：
- verdict: string（可选）
- field: string（可选）
- page: integer（可选）
- keyword: string（可选）

### 2.2 ResultViewItem

```json
{
  "rule_id": "R-RSA-001",
  "standard_id": "S1-1.2",
  "check_item": "不合适的加密算法",
  "field": "crypto.rsa.key_length",
  "value": "2048",
  "verdict": "REVIEW",
  "reason": "RSA 2048 requires review",
  "page": 3,
  "snippet": "RSA2048",
  "image_ref": "img-0001"
}
```

### 2.3 ExportPayload

```json
{
  "task_id": "task-0001",
  "format": "json",
  "include_evidence": true,
  "include_review": true
}
```

## 3. 模块接口

## 3.1 web

### list_results(task_id: str, filters: QueryFilter) -> list[ResultViewItem]

输入：
- task_id: string
- filters: QueryFilter

输出：
- list[ResultViewItem]

### get_evidence(task_id: str, rule_id: str) -> dict[str, object]

输出示例：
```json
{
  "task_id": "task-0001",
  "rule_id": "R-RSA-001",
  "evidence": [
    {
      "page": 3,
      "snippet": "RSA2048",
      "image_ref": "img-0001"
    }
  ]
}
```

### export_result(payload: ExportPayload) -> dict[str, object]

输出示例：
```json
{
  "task_id": "task-0001",
  "format": "json",
  "download_url": "/exports/task-0001.json",
  "generated_at": "2026-07-01T10:00:00Z"
}
```

## 3.2 evidence

### build_view_model(result: dict[str, object], traces: list[dict[str, object]]) -> ResultViewItem

输入：
- 单条规则结果 + 证据列表

输出：
- 可直接渲染的 ResultViewItem

### validate_trace_consistency(results: list[dict[str, object]], traces: list[dict[str, object]]) -> dict[str, object]

输出示例：
```json
{
  "ok": true,
  "missing_trace_rule_ids": [],
  "orphan_trace_rule_ids": []
}
```

## 3.3 reporting

### export_summary(results: list[dict[str, str]]) -> dict[str, int]

保持 S3 兼容。

### export_detail(results: list[ResultViewItem], summary: dict[str, int]) -> dict[str, object]

输出示例：
```json
{
  "summary": {
    "PASS": 8,
    "FAIL": 1,
    "REVIEW": 3
  },
  "items": [
    {
      "rule_id": "R-RSA-001",
      "verdict": "REVIEW"
    }
  ]
}
```

## 4. S4 主流程响应结构

```json
{
  "task": {
    "task_id": "task-0001",
    "status": "completed"
  },
  "summary": {
    "PASS": 8,
    "FAIL": 1,
    "REVIEW": 3
  },
  "results": [
    {
      "rule_id": "R-RSA-001",
      "standard_id": "S1-1.2",
      "verdict": "REVIEW",
      "page": 3,
      "snippet": "RSA2048"
    }
  ]
}
```

## 5. 兼容性要求

- S4 新增查询与导出接口，不修改 S1-S3 核心字段语义。
- 导出内容必须与页面展示一致（字段和值一致）。
- 页面上的 FAIL 只表示条款不符合，不是整份报告失败。
