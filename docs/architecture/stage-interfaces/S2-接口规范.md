# S2 接口规范（2026-04-21 至 2026-05-20）

## 1. 阶段目标

S2 目标是打通“可提取、可定位、可保存”的多格式输入链路，重点强化 parser、ocr、storage，并为 S3 的条款级判定提供稳定输入。
Word 解析不是附加项，而是与 PDF 并列的正式输入路径，必须支持正文、表格和嵌入图片的拆解。

## 2. 新增与扩展数据结构

### 2.1 ParseResultV2

```json
{
  "schema_version": "2.1",
  "task_id": "task-0001",
  "input_type": "pdf",
  "text_blocks": ["..."],
  "images": [
    {
      "image_id": "img-0001",
      "path": "tmp/page_3_img_1.png",
      "page": 3
    }
  ],
  "meta": {
    "page_count": 12,
    "parser": "pymupdf",
    "ocr_enabled": true
  }
}
```

字段类型：
- schema_version: string
- task_id: string
- input_type: string（pdf | word | scan | screenshot）
- text_blocks: list[string]
- images: list[ImageItem]
- meta.page_count: integer
- meta.parser: string
- meta.ocr_enabled: boolean

### 2.2 OCRResultV2

```json
{
  "image_id": "img-0001",
  "page": 3,
  "source_type": "image_ocr",
  "raw_text": "TLS1.2 RSA2048",
  "normalized_text": "TLS-1.2 RSA-2048",
  "confidence": 0.91,
  "tokens": ["TLS1.2", "RSA2048"]
}
```

字段类型：
- image_id: string
- page: integer
- source_type: string
- raw_text: string
- normalized_text: string
- confidence: number
- tokens: list[string]

### 2.3 StructuredField

```json
{
  "field": "crypto.rsa.key_length",
  "value": "2048",
  "source_type": "image_ocr",
  "input_type": "pdf",
  "page": 3,
  "snippet": "RSA2048",
  "confidence": 0.9,
  "source_ref": "img-0001"
}
```

字段类型：
- field: string
- value: string
- source_type: string（text | image_ocr | word_text）
- input_type: string（pdf | word | scan | screenshot）
- page: integer
- snippet: string
- confidence: number
- source_ref: string

## 3. 模块接口（S2新增契约）

## 3.1 parser

### extract_text_and_images(input_path: Path) -> ParseResultV2

输入：
- input_path: pathlib.Path

输出：ParseResultV2

异常：
- TypeError
- FileNotFoundError（建议）

实现约束：
- 当输入是 `.pdf` 时，解析器输出文本块和图片对象。
- 当输入是 `.docx` 时，解析器必须按正文段落、表格与嵌入图片拆分输出，不得把 Word 当作普通文本文件直接读取。
- Word 文本对应的 `source_type` 必须保持为 `word_text`，并与 `input_type=word` 对齐。
- Word 文档中的图片必须进入 OCR 主链路，不能被忽略。

### extract_structured_fields(parse_result: ParseResultV2, ocr_results: list[OCRResultV2]) -> list[StructuredField]

输入：
- parse_result: ParseResultV2
- ocr_results: list[OCRResultV2]

输出：list[StructuredField]

输入：
- image_path: pathlib.Path

输出：OCRResultV2

### run_batch_ocr(image_paths: list[Path]) -> list[OCRResultV2]

输入：
- image_paths: list[pathlib.Path]

输出：list[OCRResultV2]

### add_parse_record(task_id: str, record: ParseResultV2) -> None

### add_ocr_records(task_id: str, records: list[OCRResultV2]) -> None

### add_structured_fields(task_id: str, fields: list[StructuredField]) -> None

## 3.4 web

### submit_report(file_name: str, input_type: str | None = None) -> Task

扩展建议：
- 增加 task_type 字段（single | batch）
- 增加 created_at（ISO8601）
- 增加 input_type 字段（pdf | word | scan | screenshot）

### get_task(task_id: str) -> Task

扩展建议：
- status 增加 running | failed
- 增加 input_type 和 source_count

## 5. 兼容性要求

- 保持 S1 submit_report/get_task 接口可用。
- 新字段采用“可选字段”策略，不破坏旧调用方。
- S2 只输出结构化字段，不引入整份报告级别的最终结论。
