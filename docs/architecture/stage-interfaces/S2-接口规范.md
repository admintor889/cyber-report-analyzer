# S2 接口规范（段落级追溯版）

## 1. 阶段目标

S2 负责打通多格式输入和段落级来源映射。PDF、Word、扫描件和截图都要进入同一条解析链路，OCR 必须进入主流程。S2 不产出条款级结论，只产出可回放的结构化字段、页码、段落编号和来源引用。

## 2. 冻结对象

## 3. 模块接口

### 3.1 parser

#### extract_text_and_images(input_path: Path) -> ParseResultV2

- PDF 与 Word 共用入口。
- Word 必须解析正文、表格和图片，不得当成普通文本文件直接读取。
- Word 正文 `source_type` 固定为 `word_text`。
- 图片必须进入 OCR 主链路。

#### extract_structured_fields(parse_result: ParseResultV2, ocr_results: list[OCRResultV2]) -> list[StructuredField]

- 输出字段必须携带 `section_id`、`paragraph_id`、`page`、`source_ref`。

### 3.2 ocr

#### run_ocr(image_path: Path) -> OCRResultV2
#### run_batch_ocr(image_paths: list[Path]) -> list[OCRResultV2]

### 3.3 storage

#### add_parse_record(task_id: str, record: ParseResultV2) -> None
#### add_ocr_records(task_id: str, records: list[OCRResultV2]) -> None
#### add_structured_fields(task_id: str, fields: list[StructuredField]) -> None
#### get_structured_fields(task_id: str) -> list[StructuredField]

### 3.4 web

#### submit_report(file_name: str, input_type: str | None = None) -> Task

- `input_type` 取值：`pdf` | `word` | `scan` | `screenshot`
- 任务对象建议增加 `task_type`、`created_at`、`input_type`

#### get_task(task_id: str) -> Task

- 建议返回 `status`、`input_type`、`source_count`

## 4. 兼容性要求

- S1 的任务接口保留可用。
- `StructuredField` 语义不得在 S2 内被随意改写。
- S2 只输出结构化数据，不引入条款级最终结论。
- 所有结果必须能回放到段落级编号和页码。

## 5. 退出标准

S2 结束时，S3 必须可以直接消费 `StructuredField[]`，并且每个字段至少带有来源页码、段落级编号、来源类型和原始片段。
