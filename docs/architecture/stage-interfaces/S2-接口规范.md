# S2 接口规范（2026-04-21 至 2026-05-20）

## 1. 阶段目标

S2 目标是打通“可提取、可定位、可保存”的数据链路，重点强化 parser、ocr、storage，并为 S3 的条款级判定提供稳定输入。

## 2. 新增与扩展数据结构

### 2.1 ParseResultV2

```json
{
  "schema_version": "2.0",
  "task_id": "task-0001",
  "text_blocks": ["..."],
  "images": [
    {
      "image_id": "img-0001",
      "path": "tmp/page_3_img_1.png",
      "page": 3
    }
  ],
  "meta": {
    "page_count": 12,
    "parser": "pymupdf"
  }
}
```

字段类型：
- schema_version: string
- task_id: string
- text_blocks: list[string]
- images: list[ImageItem]
- meta.page_count: integer
- meta.parser: string

### 2.2 OCRResultV2

```json
{
  "image_id": "img-0001",
  "page": 3,
  "raw_text": "TLS1.2 RSA2048",
  "normalized_text": "TLS-1.2 RSA-2048",
  "confidence": 0.91,
  "tokens": ["TLS1.2", "RSA2048"]
}
```

字段类型：
- image_id: string
- page: integer
- raw_text: string
- normalized_text: string
- confidence: number
- tokens: list[string]

### 2.3 StructuredField

```json
{
  "field": "crypto.rsa.key_length",
  "value": "2048",
  "source_type": "image_ocr",
  "page": 3,
  "snippet": "RSA2048",
  "confidence": 0.9
}
```

字段类型：
- field: string
- value: string
- source_type: string（text | image_ocr）
- page: integer
- snippet: string
- confidence: number

## 3. 模块接口（S2新增契约）

## 3.1 parser

### extract_text_and_images(pdf_path: Path) -> ParseResultV2

输入：
- pdf_path: pathlib.Path

输出：ParseResultV2

异常：
- TypeError
- FileNotFoundError（建议）

### extract_structured_fields(parse_result: ParseResultV2, ocr_results: list[OCRResultV2]) -> list[StructuredField]

输入：
- parse_result: object
- ocr_results: list

输出：list[StructuredField]

## 3.2 ocr

### run_ocr(image_path: Path) -> OCRResultV2

输入：
- image_path: pathlib.Path

输出：OCRResultV2

### run_batch_ocr(image_paths: list[Path]) -> list[OCRResultV2]

输入：
- image_paths: list[pathlib.Path]

输出：list[OCRResultV2]

## 3.3 storage

### add_parse_record(task_id: str, record: ParseResultV2) -> None

### add_ocr_records(task_id: str, records: list[OCRResultV2]) -> None

### add_structured_fields(task_id: str, fields: list[StructuredField]) -> None

### get_structured_fields(task_id: str) -> list[StructuredField]

说明：
- S2 可先在 InMemoryRepository 扩展对应方法。
- S3 或 S4 再迁移 SQLite。

## 3.4 web

### submit_report(file_name: str) -> Task

扩展建议：
- 增加 task_type 字段（single | batch）
- 增加 created_at（ISO8601）

### get_task(task_id: str) -> Task

扩展建议：
- status 增加 running | failed

## 4. S2 联调输出建议

```json
{
  "task_id": "task-0001",
  "parse_result": {
    "schema_version": "2.0"
  },
  "ocr_results": [
    {
      "image_id": "img-0001",
      "page": 3
    }
  ],
  "fields": [
    {
      "field": "crypto.tls.version",
      "value": "1.2",
      "source_type": "image_ocr",
      "page": 3,
      "snippet": "TLS1.2",
      "confidence": 0.91
    }
  ]
}
```

## 5. 兼容性要求

- 保持 S1 submit_report/get_task 接口可用。
- 新字段采用“可选字段”策略，不破坏旧调用方。
- S2 只输出结构化字段，不引入整份报告级别的最终结论。
