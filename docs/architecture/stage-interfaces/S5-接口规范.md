# S5 接口规范（发布与归档版）

## 1. 阶段目标

S5 负责发布冻结和结题归档。接口重点转向稳定性、版本兼容和可追溯交付，同时保持条款级结果语义不变。

## 2. 版本化约定

### 2.1 Envelope

```json
{
  "api_version": "v1.0",
  "trace_id": "trace-20260720-0001",
  "data": {},
  "error": null
}
```

### 2.2 ErrorObject

```json
{
  "code": "TASK_NOT_FOUND",
  "message": "task not found",
  "retryable": false
}
```

### 2.3 ReleaseArtifact

```json
{
  "task_id": "task-0001",
  "version": "v1.0-final",
  "artifact": "exports/task-0001.docx",
  "checksum": "sha256:...",
  "rule_version": "2026.06"
}
```

## 3. 模块接口

### 3.1 web

#### create_task(request: dict[str, object]) -> Envelope
#### query_task(task_id: str) -> Envelope
#### export_task(task_id: str, format: str) -> Envelope
#### get_release_info(task_id: str) -> Envelope

### 3.2 storage

#### upsert_task(task: dict[str, object]) -> None
#### get_task_by_id(task_id: str) -> dict[str, object] | null
#### list_task_results(task_id: str) -> list[dict[str, object]]

### 3.3 reporting

#### build_release_report(task_id: str, detail: dict[str, object]) -> dict[str, object]

## 4. 归档要求

- 最终 Word 报告、导出结果、部署说明、演示脚本、用户手册、最终测试报告和结题纪要都要入档。
- 导出结果必须提供 checksum。
- 发布报告必须包含 `rule_version` 和 artifact 路径。

## 5. 兼容性策略

- S5 版本号固定为 `v1.0`。
- 新字段只增不删，旧字段语义不变。
- 破坏性变更必须通过下一版本发布。
