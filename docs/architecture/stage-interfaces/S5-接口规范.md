# S5 接口规范（2026-07-11 至 2026-07-31）

## 1. 阶段目标

S5 目标是发布与结题，接口重点转向稳定性、版本兼容和可追溯交付，同时保持条款级结果语义不变。

## 2. 版本化约定

### 2.1 Envelope（统一响应包）

```json
{
  "api_version": "v1.0",
  "trace_id": "trace-20260720-0001",
  "data": {},
  "error": null
}
```

字段类型：
- api_version: string
- trace_id: string
- data: object
- error: object | null

### 2.2 ErrorObject

```json
{
  "code": "TASK_NOT_FOUND",
  "message": "task not found",
  "retryable": false
}
```

字段类型：
- code: string
- message: string
- retryable: boolean

## 3. 模块接口（发布版契约）

## 3.1 web

### create_task(request: dict[str, object]) -> Envelope

请求示例：
```json
{
  "file_name": "report.pdf",
  "task_type": "single"
}
```

响应 data：Task

### query_task(task_id: str) -> Envelope

响应 data：
```json
{
  "task_id": "task-0001",
  "status": "completed",
  "summary": {
    "PASS": 8,
    "FAIL": 1,
    "REVIEW": 3
  },
  "findings_count": 12
}
```

### export_task(task_id: str, format: str) -> Envelope

输入：
- task_id: string
- format: string（json | csv）

输出 data：
```json
{
  "task_id": "task-0001",
  "format": "json",
  "artifact": "exports/task-0001.json",
  "checksum": "sha256:..."
}
```

## 3.2 storage

### upsert_task(task: dict[str, object]) -> None

### get_task_by_id(task_id: str) -> dict[str, object] | null

### list_task_results(task_id: str) -> list[dict[str, object]]

S5 要求：
- 存储层接口语义稳定，不随底层实现变化。
- 可由 InMemory 替换为 SQLite/PostgreSQL。

## 3.3 reporting

### build_release_report(task_id: str, detail: dict[str, object]) -> dict[str, object]

输出示例：
```json
{
  "task_id": "task-0001",
  "generated_at": "2026-07-29T12:00:00Z",
  "version": "v1.0-final",
  "summary": {
    "PASS": 8,
    "FAIL": 1,
    "REVIEW": 3
  },
  "artifacts": [
    "exports/task-0001.json"
  ]
}
```

## 4. 回归与验收接口要求

- 所有对外响应建议采用 Envelope 包装。
- 所有导出结果必须提供 checksum（建议）。
- 任务查询必须可返回 trace_id，便于问题追踪。

## 5. 兼容性策略

- S5 版本号固定为 v1.0。
- 新字段只增不删，旧字段语义不变。
- 破坏性变更必须通过 v2 路径发布。
