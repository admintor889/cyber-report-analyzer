# 网络安全报告分析工具 模块接口文档 v1

## 1. 文档范围

本文档定义当前代码中各板块接口约定，覆盖以下模块：
- 解析模块 parser
- OCR模块 ocr
- 规则引擎模块 rules_engine
- 模型复核模块 model_review
- 证据溯源模块 evidence
- 报告输出模块 reporting
- 后端任务模块 web
- 存储模块 storage

说明：
- 本文档以当前仓库实现为基线，字段可在后续版本扩展。
- 所有示例默认使用 UTF-8 编码。

## 2. 全局数据类型约定

### 2.1 基础枚举

- Verdict：PASS | FAIL | REVIEW（条款级）
- TaskStatus：queued | completed
- SourceType（建议）：text | image_ocr

### 2.2 通用对象

1. Task 对象
- task_id: string，任务唯一标识，例如 task-0001
- file_name: string，提交的报告文件名
- status: string，任务状态，当前支持 queued/completed
- result: object|null，任务结果

2. RuleResult 对象（建议）
- rule_id: string，规则编号
- standard_id: string，标准编号
- check_item: string，检查项名称
- verdict: string，PASS/FAIL/REVIEW（仅表示对应条款结果）
- reason: string，命中原因
- field: string，命中字段
- value: string，命中值

3. EvidenceTrace 对象
- field: string，字段名
- value: string，字段值
- page: string，页码（当前为字符串）
- snippet: string，证据片段
- image_ref: string，截图引用路径或ID

## 3. 模块接口定义

## 3.1 parser 模块

文件：src/parser/pdf_parser.py

接口：extract_text_and_images(pdf_path)

输入：
- pdf_path: pathlib.Path
- 要求：必须为 Path 类型

输出：
- 类型：dict[str, list[str]]
- 字段：
  - text_blocks: list[string]，正文分段列表
  - image_paths: list[string]，截图路径列表

异常：
- TypeError：当 pdf_path 不是 pathlib.Path

示例输入：
- pdf_path = Path("samples/input-pdf/demo.pdf")

示例输出：
{
  "text_blocks": ["TLS version is 1.2", "RSA2048 configured"],
  "image_paths": ["tmp/page_3_img_1.png", "tmp/page_8_img_2.png"]
}

## 3.2 ocr 模块

文件：src/ocr/ocr_pipeline.py

接口：run_ocr(image_path)

输入：
- image_path: pathlib.Path
- 要求：必须为 Path 类型

输出：
- 类型：dict[str, str]
- 字段：
  - raw_text: string，OCR原始文本
  - normalized_text: string，标准化后文本

异常：
- TypeError：当 image_path 不是 pathlib.Path

示例输出：
{
  "raw_text": "Protocol: TLS1.2",
  "normalized_text": "Protocol TLS-1.2"
}

## 3.3 rules_engine 模块

文件：src/rules_engine/rule_engine.py

接口：evaluate_rules(fields, rules)

输入：
- fields: dict[str, str]
  - 示例：{"crypto.rsa": "2048", "crypto.tls": "1.2"}
- rules: list[dict[str, str]]
  - 示例：[{"rule_id": "R-RSA-001", "field": "crypto.rsa", "op": ">=", "value": "3072"}]

输出：
- 类型：list[dict[str, str]]
- 建议每项包含：rule_id、verdict、reason、field、value
- 当前实现返回空列表占位

异常：
- TypeError：fields 不是 dict
- TypeError：rules 不是 list

## 3.4 model_review 模块

文件：src/model_review/reviewer.py

接口：semantic_review(item)

输入：
- item: dict[str, str]
- 支持字段优先级：value > raw_text > text

输出：
- 类型：dict[str, str]
- 字段：
  - normalized: string，归一化结果
  - explanation: string，解释信息
  - confidence: string，置信度（0~1 的字符串）

当前规则能力：
- RSA识别：支持 RSA2048、2048-bit RSA 等
- TLS识别：支持 TLS1.0/TLS1.1/TLS1.2/TLS1.3
- 弱算法关键词：MD5、SHA-1、DES、3DES、RC4、ECB
- 未命中规则：返回原文本并提示人工复核

异常：
- TypeError：item 不是 dict

示例输入：
{"value": "RSA2048"}

示例输出：
{
  "normalized": "RSA-2048",
  "explanation": "RSA key length meets minimum baseline but should be reviewed by policy.",
  "confidence": "0.90"
}

## 3.5 evidence 模块

文件：src/evidence/tracer.py

接口：build_evidence_trace(hit)

输入：
- hit: dict[str, str]
- 建议字段：field、value、page、snippet、image_ref

输出：
- 类型：dict[str, str]
- 固定字段：field、value、page、snippet、image_ref
- 当前实现返回空字符串占位结构

异常：
- TypeError：hit 不是 dict

## 3.6 reporting 模块

文件：src/reporting/report_exporter.py

接口：export_summary(results)

输入：
- results: list[dict[str, str]]
- 每项建议包含 verdict 字段

输出：
- 类型：dict[str, int]
- 字段：
  - PASS: int
  - FAIL: int
  - REVIEW: int

规则：
- 若 verdict 缺失或未知，计入 REVIEW

示例输入：
[
  {"verdict": "PASS"},
  {"verdict": "FAIL"},
  {"verdict": "UNKNOWN"}
]

示例输出：
{
  "PASS": 1,
  "FAIL": 1,
  "REVIEW": 1
}

## 3.7 web 模块

文件：src/web/app.py

接口一：health()
- 输入：无
- 输出：dict[str, str]
- 输出示例：{"status": "ok"}

接口二：submit_report(file_name)
- 输入：
  - file_name: string，非空
- 输出：Task 对象
- 异常：
  - ValueError：file_name 为空或非字符串

接口三：get_task(task_id)
- 输入：
  - task_id: string
- 输出：Task 对象
- 异常：
  - KeyError：task_id 不存在

接口四：set_task_result(task_id, result)
- 输入：
  - task_id: string
  - result: dict[string, any]
- 输出：更新后的 Task 对象
- 状态变化：queued -> completed

Task 输出示例：
{
  "task_id": "task-0001",
  "file_name": "sample-report.pdf",
  "status": "completed",
  "result": {
    "summary": "ok"
  }
}

## 3.8 storage 模块

文件：src/storage/repository.py

类：InMemoryRepository

方法一：add(item)
- 输入：item: dict[str, str]
- 输出：None

方法二：all()
- 输入：无
- 输出：list[dict[str, str]]

说明：
- 当前为内存仓库，仅用于开发阶段。
- 后续可替换为 SQLite 实现，接口语义保持一致。

## 4. 跨模块数据流（建议）

标准流转顺序：
1. web.submit_report 生成任务
2. parser.extract_text_and_images 提取正文与截图
3. ocr.run_ocr 处理截图文字
4. model_review.semantic_review 进行语义归一化
5. rules_engine.evaluate_rules 输出规则判定
6. evidence.build_evidence_trace 绑定证据
7. reporting.export_summary 生成摘要
8. web.set_task_result 回填任务结果

## 5. 最小联调结果结构建议

建议 set_task_result 的 result 字段采用以下结构：
