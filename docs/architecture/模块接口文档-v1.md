# 网络安全报告分析工具 模块接口文档

## 1. 文档范围

本文档定义 S2-S5 的统一模块接口约定，覆盖 parser、ocr、rules_engine、model_review、evidence、reporting、web 和 storage。这里的目标不是描述某个阶段的临时实现，而是定义跨阶段保持稳定的数据对象和接口语义。

## 2. 全局数据类型约定

### 2.1 核心枚举

- Verdict：PASS | FAIL | REVIEW
- InputType：pdf | word | scan | screenshot
- SourceType：text | word_text | image_ocr
- TaskStatus：queued | running | completed | failed

## 3. 模块接口定义

#### extract_text_and_images(input_path: Path) -> ParseResultV2

- 支持 PDF 与 Word。
- Word 必须保留正文、表格和图片结构。
- 输出必须包含章节和段落定位。

## 4. 跨模块数据流

标准流转顺序：
1. web.submit_report 生成任务。
2. parser.extract_text_and_images 解析正文、表格和图片。
3. ocr.run_ocr 处理图片文字。
4. parser.extract_structured_fields 输出 `StructuredField[]`。
5. rules_engine.evaluate_rules 输出 `RuleResult[]`。
6. rules_engine.build_findings 生成 `Finding[]`。
7. evidence 绑定页码、段落编号、片段和图片来源。
8. reporting 输出摘要、分析报告和发布物。
9. web 回填任务结果并支持展示或导出。

## 5. 冻结规则

- `section_id`、`paragraph_id`、`page`、`source_ref` 必须跨阶段保持一致。
- `PASS / FAIL / REVIEW` 只作用于条款级结果。
- Word 导出必须复用页面 view model。
- 发布版响应建议统一 Envelope 包装。
