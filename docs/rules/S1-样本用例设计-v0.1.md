# S1 样本用例设计 v0.1

## 1. 目标

本文件用于补齐 S1 阶段字段映射、标准表、检查项与规则样本设计，覆盖 `RSA`、`TLS`、弱算法关键词三类 P0 规则，并为 S2 的结构化字段抽取提供输入样例。

设计原则：
- 每条样本必须能追踪到 S1 规则台账。
- 每条样本必须写清输入文本、期望字段、期望值和期望判定。
- 样本优先覆盖 P0 规则，P1 仅做文档化占位。
- 样本可以来自 PDF 正文或 OCR 文本，S1 不要求真实 OCR 准确率验证。

## 2. 字段约定

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| `crypto.rsa.key_length` | RSA 密钥长度，统一保存数字字符串 | `2048` |
| `crypto.tls.version` | TLS/SSL 协议版本，统一保存版本号字符串 | `1.2` |
| `crypto.weak` | 弱算法命中关键词，多个值用逗号分隔 | `MD5,RC4` |
| `source_type` | 来源类型 | `text` / `image_ocr` |
| `page` | 来源页码 | `3` |
| `snippet` | 可复现片段 | `TLS1.2 RSA2048` |

## 5. 回归使用方式

回归时建议至少抽取以下组合：
- RSA：`S1-TC-001`、`S1-TC-002`、`S1-TC-003`。
- TLS：`S1-TC-008`、`S1-TC-009`、`S1-TC-010`、`S1-TC-011`。
- 弱算法：`S1-TC-014` 至 `S1-TC-020`。
- 混合样本：`S1-TC-021` 至 `S1-TC-024`。

Issue 关闭前必须说明：
- 是否覆盖对应字段映射。
- 是否与 `field_mapping-v0.1.yaml` 字段名一致。
- 是否影响 S2 的 `StructuredField` 输入。
- 是否影响 S3 的条款级 `RuleResult` 或 `finding` 结构。
