# S5 协同开发执行手册（发布冻结版）

## 1. 阶段定位

S5 是发布收敛和结题交付阶段。目标不是继续堆功能，而是冻结版本、修阻断缺陷、补齐文档、完成演示和答辩材料，并保持条款级结果语义稳定。

## 2. 入口条件

- S4 主流程已完成彩排。
- 发布候选风险清单已列出。
- 至少一份示例 Word 报告可完整跑通。
- 测试、部署、用户手册责任人明确。

## 3. 范围边界

必须完成：
- 发布候选分支。
- 最终回归测试。
- 用户手册。
- 部署说明。
- 演示脚本和答辩材料。
- 示例 Word 报告和导出样例。
- 版本归档和最终验收纪要。

明确不做：
- 不新增高风险功能。
- 不重构核心链路。
- 不调整已冻结接口。
- 不引入新外部依赖，除非解决阻断问题。

## 4. 发布策略

- 从稳定集成分支创建 `release/v1.0-final`。
- 只允许阻断缺陷、文档、演示脚本、发布包相关变更进入 release。
- 非阻断优化记录到后续版本，不进入 S5 发布范围。

## 5. 质量门禁

- `pytest -q` 通过。
- 端到端演示脚本通过。
- 示例 Word 报告可跑通。
- 导出文件可打开，字段和页面一致。
- 部署说明可按步骤复现。
- 用户手册覆盖上传、查看、筛选、证据、导出。
- 发布材料不得把单条 `FAIL` 表述成整份报告失败。

## 6. 退出标准

S5 完成后必须归档发布标签和提交号、可运行源码和依赖说明、Word 报告和导出结果、测试报告、用户手册、部署说明、答辩材料以及最终验收纪要。
