# S4 协同开发执行手册（Word 输出一致性版）

## 1. 阶段定位

S4 是 Web 集成和规则维护阶段。目标是把 S3 的分析结果接入轻量 Web 流程，让用户能完成上传、查看、筛选、证据查看、规则维护和 Word 导出，并保证页面与 Word 输出一致。

## 2. 入口条件

- S3 主链路结果结构已冻结。
- `RuleResult`、`ReviewItem`、`EvidenceTraceV3` 可稳定生成。
- 至少有一组可演示样本。
- 规则和模型回归集可运行。

## 3. 范围边界

必须完成：
- 上传和任务查询流程。
- 结果列表、筛选、详情查看。
- 证据查看。
- 规则维护界面和立即生效流程。
- Word 导出。
- 主流程端到端回归。

明确不做：
- 不做复杂权限角色。
- 不做企业级多租户。
- 不做大屏展示。
- 不做高并发部署。

## 4. 演示主流程

1. 上传或选择样本报告。
2. 查看任务状态。
3. 查看 summary。
4. 按 `FAIL/REVIEW` 筛选问题项。
5. 打开某条结果的证据页码、段落编号和片段。
6. 在 Web 中调整一条规则并保存。
7. 刷新结果，确认规则维护立即生效。
8. 导出 Word 结果。
9. 展示 Word 与页面内容一致。

## 5. 质量门禁

- 交互或展示变更必须附截图、录屏或接口响应样例。
- 导出字段必须和页面展示字段一致。
- Web 返回结构不能破坏 S3 核心字段。
- 页面上的 `FAIL` 只表示条款不符合，不表示整份报告失败。
- 规则保存后必须立即可见，且版本号可追踪。

## 6. 退出标准

S4 完成后必须交给 S5 可运行演示路径、可导出的示例结果、缺陷闭环清单、部署和联调说明、发布候选风险清单和规则维护操作记录。
