# S2 协同开发执行手册（段落级追溯版）

## 1. 阶段定位

S2 是多格式输入和结构化抽取阶段。目标是把 PDF、Word、扫描件和截图统一接入同一条解析链路，并把命中点定位到段落级编号、页码和来源引用。

## 2. 入口条件

- S1 基线测试通过。
- `ParseResultV2`、`OCRResultV2`、`StructuredField` 的语义已经确认。
- 至少准备 5 份样本，且样本里同时覆盖文本页和图片页。
- 本地环境和 CI 可用。

## 3. 范围边界

必须完成：
- PDF 和 Word 的正文提取、页码映射和图片分离。
- OCR 单张和批量接口。
- OCR 后处理和字段清洗。
- 结构化字段输出与来源记录保存。
- 段落级编号、章节序号和多溯源点保存。

明确不做：
- 不做条款级最终结论。
- 不做完整 Word 报告导出。
- 不做复杂 UI。
- 不做大规模性能优化。

## 4. 核心交付

- 段落级来源映射规范。
- 结构化字段协议。
- 解析、OCR 和字段回放回归测试。
- 样本输入与输出说明。

## 5. 联调主线

1. 上传或指定 PDF/Word。
2. parser 输出 `ParseResultV2`。
3. OCR 对图片输出 `OCRResultV2`。
4. 字段抽取合并正文、Word 文本和 OCR，输出 `StructuredField[]`。
5. storage 保存 parse、OCR、field 记录。
6. web 查询任务时可看到解析状态、输入类型、字段数量和段落编号。

## 6. 质量门禁

- parser/OCR 变更必须包含样本或构造测试。
- 字段名变更必须同步更新字段映射表。
- 存储接口必须保持调用方兼容。
- 批处理失败必须有错误记录，不能静默丢失。
- 字段语义必须保持 `field/value/source_type/page/section_id/paragraph_id/snippet/confidence/input_type` 一致。

## 7. 退出标准

S2 完成后必须交给 S3 一个稳定输入：`StructuredField[]` 可直接进入规则引擎，每个字段都带来源页码、段落级编号、来源类型和原始片段。
