## 2. GitHub协作基线（S1必须遵守）

分支规则：
- 主分支：main（只放可发布内容）
- 集成分支：develop（S1统一集成）
- 功能分支：feature/模块-姓名-功能

提交与评审规则：
- 禁止直接推送 main
- 所有开发通过 PR 合并到 develop
- 每个 PR 至少 1 人评审
- 核心模块 parser、ocr、rules、model 至少 2 人评审
- PR 必须通过 CI

- feature/rules-qiuxintian-s1-mapping

## 5. Issue与PR最小模板要求

- 输入输出定义

- 截止日期

- 测试证据

PR大小建议：
- 单个 PR 控制在 200 至 500 行有效变更
- 仅解决 1 至 3 个逻辑点
