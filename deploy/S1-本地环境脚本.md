# S1 本地环境脚本说明

负责人：叶泽东  
阶段：S1  
目标：让新成员可以在本地快速完成最小开发环境初始化，并验证 parser MVP 能正常开发与测试。

## 1. 交付文件

- 依赖清单：`requirements.txt`
- 脚本：`scripts/setup_local_env.py`
- 本说明：`deploy/S1-本地环境脚本.md`

## 2. 脚本能力

脚本默认完成以下动作：

1. 检查本机 Python
2. 创建或复用 `.venv`
3. 升级 `pip`
4. 按 `requirements.txt` 安装固定依赖版本
5. 输出虚拟环境激活方式

可选动作：

- 执行 `python -m compileall src tests`
- 执行 `python -m pytest -q`

## 3. 使用方式

在项目根目录执行：

```bash
python scripts/setup_local_env.py
```

脚本会自动读取：

```text
requirements.txt
```

并安装其中定义的依赖版本，以保证团队本地开发环境尽量一致。

如果要在初始化后立即做一次检查：

```bash
python scripts/setup_local_env.py --run-checks
```

如果想自定义虚拟环境目录：

```bash
python scripts/setup_local_env.py --venv-path .venv-s1
```

## 4. 初始化完成后的常用命令

激活虚拟环境：

```bash
.venv\Scripts\python.exe -m compileall src tests
```

执行测试：

```bash
.venv\Scripts\python.exe -m pytest -q
```

## 5. 适用范围

本脚本用于 S1 最小 MVP 开发环境，不处理以下内容：

- 生产部署
- Docker / 容器化
- 数据库初始化
- Web 服务上线

## 6. 验收标准

满足以下条件即可视为完成 S1 本地开发环境脚本初版：

- [ ] 新成员可一键创建虚拟环境
- [ ] 依赖可从 `requirements.txt` 自动安装
- [ ] 可选执行基础检查
- [ ] 输出明确的下一步操作提示
